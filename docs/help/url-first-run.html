üéØ First Run ‚Äî Stop Burning Tokens

Your bot just came alive and wants to have a long intro conversation. That's expensive. Here's how to skip the expensive part and get straight to using it.

‚ö†Ô∏è The Problem: The intro/bootstrap conversation can burn 15-20k+ tokens just getting to know you. That's $0.50-$1.00+ on Sonnet before you've even done anything useful. Every session without memory files repeats this.
Step 1 ‚Äî Stop the current conversation

If you're in a chat right now, type:

/new

This starts a fresh session. The old one's tokens are already spent ‚Äî don't keep adding to it.

Step 2 ‚Äî Create your memory files

These files tell your bot who you are so it doesn't have to ASK (and burn tokens doing it).

Create the workspace and memory folder:

mkdir -p ~/.openclaw/workspace/memory

Create USER.md ‚Äî info about you:

cat > ~/.openclaw/workspace/USER.md << 'EOF'
# About Me
- Name: (your name)
- What to call me: (nickname or first name)
- Timezone: America/New_York
- Interests: (list a few things you're into)
- What I want help with: (what you plan to use the bot for)

## Notes
- (anything else you want it to know about you)
EOF
Edit this file and replace the (parentheses) parts with your actual info. Use any text editor: nano ~/.openclaw/workspace/USER.md

Create IDENTITY.md ‚Äî who your bot is:

cat > ~/.openclaw/workspace/IDENTITY.md << 'EOF'
# Identity
- Name: (pick a name for your bot)
- Personality: helpful, casual, to the point
- Emoji: ü§ñ
EOF

Create SOUL.md ‚Äî behavior rules:

cat > ~/.openclaw/workspace/SOUL.md << 'EOF'
# Behavior
- Be helpful and concise
- Don't waste tokens on filler ("Great question!" etc.)
- Remember things by writing to memory files
- Ask before doing anything that affects the outside world
EOF
Step 3 ‚Äî Delete the bootstrap file

The file BOOTSTRAP.md tells your bot to have a long getting-to-know-you conversation every time. Once you've set up USER.md and IDENTITY.md, you don't need it anymore.

rm ~/.openclaw/workspace/BOOTSTRAP.md 2>/dev/null; echo "Done"
‚úÖ Without BOOTSTRAP.md, your bot skips the intro and just starts working. It reads USER.md and IDENTITY.md instead ‚Äî way cheaper.
Step 4 ‚Äî Start a fresh conversation

Now start a new chat (via Telegram or the web dashboard). Your bot should:

  • Already know your name (from USER.md)
  • Already know its own name (from IDENTITY.md)
  • Skip the long intro
  • Just be ready to help

Try saying something like: "Hey, what do you know about me?"

It should pull from your USER.md file. If it does ‚Äî you're set.

Step 5 ‚Äî Ongoing token-saving habits

Start fresh often

Use /new when you switch topics. Long conversations get expensive because the ENTIRE history gets resent with every message.

Let the bot use files

Instead of pasting huge blocks of text in chat, save it to a file and tell the bot to read it:

  • Bad: (paste 5000 words into chat) "fix this"
  • Good: "read ~/myfile.txt and fix the bug on line 42"

Tell it to remember things

Say "remember that I prefer X" or "save this for later" ‚Äî it'll write to memory files instead of you having to repeat yourself next session.

Check your spend

Visit console.anthropic.com/settings/billing after your first day to see what the intro cost vs. normal use.

With memory files: Bot reads USER.md on startup. Already knows you. Straight to work. Cheap.
