Phase 1
Install Linux

If you're already on Linux, skip to Phase 2.

Option A: Ubuntu Desktop (Recommended for beginners)

  1. Download Ubuntu Desktop from ubuntu.com/download/desktop
  2. Create a bootable USB with Rufus (Windows) or Balena Etcher
  3. Boot from USB (usually F12 or F2 at startup to pick boot device)
  4. Choose "Install Ubuntu" ‚Äî use the guided installer
  5. When asked about disk: "Erase disk and install Ubuntu" (if replacing Windows entirely)
  6. Set your username and password ‚Äî remember these, you'll need them constantly
  7. Reboot when done, remove USB

Option B: Linux Mint (Even easier, more Windows-like)

  1. Download from linuxmint.com ‚Äî pick Cinnamon edition
  2. Same USB creation + boot process as above
‚úÖ Once you're at a desktop and can open a terminal (Ctrl+Alt+T), you're ready for Phase 2.

Phase 2
Create Bot Accounts

‚ö†Ô∏è Do this BEFORE installing anything. Your bot gets its OWN accounts. Never use your personal ones.

1. Create a new Google account for your bot

  1. Go to accounts.google.com/signup
  2. Pick a name like yourbot.name@gmail.com
  3. This will be used for: Google Drive, YouTube, Gmail (later)
  4. Write down the email and password somewhere safe

2. Create a new GitHub account for your bot

  1. Go to github.com/signup
  2. Use the bot's Gmail address
  3. Pick a username for your bot
  4. This will be used for: repos, GitHub Pages, automation

3. Create an Anthropic account (for the API)

  1. Go to console.anthropic.com
  2. Sign up with the bot's Gmail address
  3. You'll add payment and get your API key in Phase 4

4. Create a Telegram bot

  1. Open Telegram on your phone
  2. Search for @BotFather and start a chat
  3. Send: /newbot
  4. Give it a display name (e.g. "My Bot")
  5. Give it a username (e.g. mybot_ai_bot ‚Äî must end in "bot")
  6. BotFather gives you a token ‚Äî looks like 7123456789:AAH...
  7. Save this token. You'll need it in Phase 9.
‚úÖ You should now have: bot Gmail, bot GitHub, Anthropic account, Telegram bot token. All separate from your personal stuff.

Phase 3
Install OpenClaw

Open a terminal (Ctrl+Alt+T) and run these one at a time:

1. Update your system first

sudo apt update && sudo apt upgrade -y

2. Install Node.js (OpenClaw needs it)

curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs

Verify it worked:

node --version
Should show v22.x.x or higher. If it shows an old version, something went wrong ‚Äî ask Ryan.

3. Install Git

sudo apt install -y git

4. Install OpenClaw

npm install -g openclaw

Verify it installed:

openclaw help
‚úÖ If you see OpenClaw's help text with commands listed, it's installed. If you get "command not found", try: sudo npm install -g openclaw

Phase 4
Connect Your AI Brain (Anthropic API)

1. Add a payment method

  1. Go to console.anthropic.com/settings/billing
  2. Add your credit/debit card
  3. Add $50 in credits (your first two-week budget)

2. Set a hard spending limit

‚ö†Ô∏è DO THIS NOW. This is your #1 safety net against surprise bills.
  1. Go to console.anthropic.com/settings/limits
  2. Set "Monthly spend limit" to $100 (or $50 if you want to be extra safe)
  3. The API will literally stop working when you hit this cap. No surprises.

3. Create an API key

  1. Go to console.anthropic.com/settings/keys
  2. Click "Create Key"
  3. Name it something like "openclaw"
  4. Copy the key ‚Äî it starts with sk-ant-...
  5. Save it somewhere safe. You can't see it again after closing the page.
‚úÖ You now have an API key with a spending cap. Your wallet is protected.

Phase 6
Security Hardening

Your bot runs on your machine. Lock it down so nothing from outside can reach it.

Skip: Active Directory

You do NOT need Active Directory. AD is for corporate/enterprise networks with domain controllers and managed fleets of computers. For a personal OpenClaw setup, the firewall + SSH hardening below is the right level of security.

1. Enable the firewall

sudo ufw enable
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow ssh

2. Only allow the gateway on your local network

Replace 192.168.x.0/24 with your actual LAN subnet. If your IP is 192.168.1.50, your subnet is 192.168.1.0/24.
sudo ufw allow from 192.168.1.0/24 to any port 18789

Check your IP to figure out your subnet:

hostname -I

3. Harden SSH

sudo sed -i 's/#PermitRootLogin yes/PermitRootLogin no/' /etc/ssh/sshd_config
sudo systemctl restart sshd

4. Enable automatic security updates

sudo apt install -y unattended-upgrades
sudo dpkg-reconfigure -plow unattended-upgrades

Select Yes when prompted.

5. Set your timezone

sudo timedatectl set-timezone America/New_York
Change to your timezone if different. List all: timedatectl list-timezones
‚úÖ Machine is locked down. Firewall up, SSH hardened, auto-updates on.

Phase 7
Token Saving & Memory Optimization

This is the difference between burning through $100 in a week vs making it last the whole month.

1. Open your config

openclaw config edit

2. Make sure these are in your config

Find the "agents" section and make sure it includes:

Default model (Sonnet = best value)

Under agents.defaults.model:

"model": {
  "primary": "anthropic/claude-sonnet-4-20250514"
}

Compaction (auto-summarizes old conversation)

Under agents.defaults:

"compaction": {
  "mode": "safeguard",
  "memoryFlush": {
    "enabled": true
  }
}
This is crucial. Without compaction, every message resends your ENTIRE conversation history. With it, old messages get summarized, saving massive tokens.

Memory search (persistent knowledge across sessions)

"memorySearch": {
  "enabled": true,
  "sources": ["memory", "sessions"]
}

3. Create your workspace memory structure

mkdir -p ~/.openclaw/workspace/memory

Your bot will use these files to remember things across sessions:

  • MEMORY.md ‚Äî long-term curated knowledge (the bot writes this)
  • memory/YYYY-MM-DD.md ‚Äî daily notes
  • USER.md ‚Äî info about you (so it doesn't ask every session)
  • IDENTITY.md ‚Äî who your bot is (name, personality)
  • SOUL.md ‚Äî behavior rules and personality
  • TOOLS.md ‚Äî local tool notes and configs
These files load as project context ‚Äî way cheaper than re-explaining things in chat. The bot reads them on startup instead of you burning tokens repeating yourself.

4. Create a basic USER.md to start

cat > ~/.openclaw/workspace/USER.md << 'EOF'
# About Me
- Name: (your name)
- Timezone: America/New_York
- Interests: (whatever you want)
EOF
Edit this file to add more about yourself. The more your bot knows from the file, the less you have to explain in chat (saving tokens).

5. Token-saving habits

  • Start fresh conversations often ‚Äî use /new when switching topics. Long conversations cost more per message because the whole history gets resent.
  • Be specific ‚Äî "fix the bug on line 42" costs less than "something is wrong with my code, can you look at everything"
  • Let the bot use files ‚Äî instead of pasting huge text blocks in chat, save to a file and tell the bot to read it
‚úÖ Token optimization done. Your $100/month will stretch way further now.

Phase 8
Permissions & Tool Access

‚ö†Ô∏è These give your bot real power on your machine. Read what each one does. Start with only what you need.

Open your config:

openclaw config edit

Understanding the tools

üñ•Ô∏è exec ‚Äî Run shell commands

What: Lets the bot run terminal commands ‚Äî install packages, run scripts, manage files.

Risk: HIGH. Full terminal access as your user.

Modes:

  • "full" ‚Äî bot can run anything
  • "allowlist" ‚Äî only pre-approved commands
  • "deny" ‚Äî no commands at all
Start with "allowlist" or "deny" until you trust your setup. You can always open it up later.

üìÅ read / write / edit ‚Äî File access

What: Read, create, and modify files in the workspace.

Risk: LOW-MEDIUM. Limited to workspace by default (~/.openclaw/workspace/).

Needed for: Memory files, notes, projects. You want this on.

üåê web_search / web_fetch ‚Äî Internet access

What: Search the web and fetch content from URLs.

Risk: LOW. Read-only. Can look things up, can't post or log in anywhere.

Needed for: Research, checking docs, pulling info.

üåç browser ‚Äî Browser automation

What: Control a web browser ‚Äî click, type, navigate, screenshot.

Risk: MEDIUM-HIGH. Can interact with websites. Only enable if you need it.

Skip this for now unless you have a specific use case.

‚è∞ cron ‚Äî Scheduled tasks & reminders

What: Set up recurring tasks ‚Äî reminders, daily checks, automated routines.

Risk: LOW. Just triggers messages/tasks on a schedule.

Needed for: Morning briefings, reminders, routine checks. Nice to have.

üí¨ message ‚Äî Send messages to channels

What: Bot can proactively send messages to Telegram without you asking first.

Risk: MEDIUM. Bot can message you (or groups) on its own.

Needed for: Reminders, alerts, cron task results. Works with Telegram.

Recommended starter setup

For your first week, keep it simple:

  • ‚úÖ read/write/edit ‚Äî on (needed for memory)
  • ‚úÖ web_search/web_fetch ‚Äî on (research)
  • ‚úÖ cron ‚Äî on (reminders)
  • ‚úÖ message ‚Äî on (Telegram proactive messages)
  • ‚ö†Ô∏è exec ‚Äî start with "allowlist" or "deny"
  • ‚ùå browser ‚Äî off until needed

You can expand permissions anytime as you get comfortable.

‚úÖ Permissions set. Your bot has the access it needs without giving away the keys to the castle.

Phase 9
Connect Telegram

Remember that bot token from Phase 2? Time to use it.

1. Open your config

openclaw config edit

2. Add the Telegram channel

Add this to your config (or find the existing channels section and add to it):

"channels": {
  "telegram": {
    "enabled": true,
    "botToken": "YOUR_BOT_TOKEN_HERE",
    "dmPolicy": "allowlist",
    "allowFrom": ["YOUR_TELEGRAM_USER_ID"]
  }
}

3. Find your Telegram user ID

  1. Open Telegram
  2. Search for @userinfobot
  3. Start a chat with it ‚Äî it'll reply with your user ID (a number like 123456789)
  4. Put that number in the "allowFrom" array above
‚ö†Ô∏è The allowFrom list is critical. Without it, ANYONE who finds your bot can talk to it and use your API credits. Only put YOUR user ID in there.

4. Restart OpenClaw

openclaw gateway restart

5. Test it

  1. Open Telegram
  2. Search for your bot by its username
  3. Send it a message: "Hey, are you alive?"
  4. It should respond
‚úÖ You can now talk to your bot from your phone, anywhere. Welcome to the future.

Phase 10
Final Checks

1. Verify everything is running

2. Check the web dashboard

http://127.0.0.1:18789

3. Test a full conversation

Send your bot a few messages via Telegram. Try:

  • "What's your name?"
  • "Remember that my favorite color is blue" (tests memory)
  • "What's my favorite color?" (in a new session ‚Äî tests if memory persists)

4. Check your API spend

5. Set OpenClaw to start on boot (optional)

So your bot starts automatically when your computer turns on:

This creates a systemd service so OpenClaw runs in the background and survives reboots.
