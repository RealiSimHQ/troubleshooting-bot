üîí Security

1. Update your system

sudo apt update && sudo apt upgrade -y

2. Enable the firewall

sudo ufw enable
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow ssh

3. Only allow the gateway on your local network

Replace 192.168.x.0/24 with your actual LAN subnet.
sudo ufw allow from 192.168.1.0/24 to any port 18789

4. Harden SSH

sudo sed -i 's/#PermitRootLogin yes/PermitRootLogin no/' /etc/ssh/sshd_config
sudo systemctl restart sshd

5. Enable automatic security updates

sudo apt install -y unattended-upgrades
sudo dpkg-reconfigure -plow unattended-upgrades

Select Yes when prompted.

6. Set your timezone

sudo timedatectl set-timezone America/New_York
Change America/New_York to your timezone if different.
‚ö° Token Saving & Memory Optimization

7. Enable prompt caching

This saves massive tokens on repeated context. Open your config:

openclaw config edit

Make sure you have prompt caching enabled in your provider settings. For Anthropic:

"promptCaching": true

8. Set up context compression

Keeps your context window from blowing up. In your gateway config, enable compaction:

"compaction": { "enabled": true }
This automatically summarizes old conversation when context gets large, saving tokens on every message.

9. Use memory files instead of relying on context

Create a workspace with memory files so your agent persists knowledge across sessions without burning context tokens:

mkdir -p ~/.openclaw/workspace/memory

Key files to create:

  • MEMORY.md ‚Äî long-term curated knowledge
  • memory/YYYY-MM-DD.md ‚Äî daily notes (raw logs)
  • USER.md ‚Äî info about you
  • IDENTITY.md ‚Äî who your agent is
  • SOUL.md ‚Äî personality & behavior rules
  • TOOLS.md ‚Äî local tool notes & configs
These files get loaded as project context instead of filling up your chat history. Much cheaper than re-explaining things every session.

10. Set a default model

If you're running Ollama (local models, no API costs):

"defaultModel": "ollama/llama3.1"

Check what models you have installed:

ollama list

Pull a good general-purpose model if you don't have one:

ollama pull llama3.1
Ollama runs 100% local ‚Äî no API keys needed, no costs. Performance depends on your hardware (GPU/RAM). Bigger models = smarter but slower.

If you switch to a paid API later (Anthropic, OpenAI, etc.), you can change this anytime.

üîë API Keys & Permissions

11. Create SEPARATE accounts for your bot

‚ö†Ô∏è IMPORTANT: Do NOT use your personal accounts. Create brand new, dedicated accounts for your bot. This keeps your personal stuff safe and gives the bot its own identity.
  • New Google account ‚Äî for Google Drive, YouTube, Gmail (e.g. yourbot.name@gmail.com)
  • New GitHub account ‚Äî for repos, Pages, automation
  • Separate API keys ‚Äî never share keys between your personal use and the bot

Then share specific folders/repos from your personal accounts TO the bot's accounts as needed. Bot gets access, you stay in control.

12. API keys (only if using paid models)

If you're running Ollama only, skip this ‚Äî you don't need any API keys. Ollama is fully local.

If you ever want to use a cloud model (Anthropic, OpenAI, etc.), you'd add the key to your gateway config at that point. For now, Ollama is all you need.

13. Give your agent tool access

‚ö†Ô∏è These give your bot real power on your machine. Read what each one does before enabling. Start with only what you need ‚Äî you can always add more later.

Open your config:

openclaw config edit

Tools are managed in the config file. Here's what each one does and how to enable it:

üñ•Ô∏è exec ‚Äî Run shell commands

What it does: Lets the bot run terminal commands on your machine ‚Äî install packages, run scripts, manage files, anything you can do in a terminal.

‚ö†Ô∏è Risk: HIGH. This is full terminal access. The bot can run any command as your user.

How to control it: In your config under the agent section, exec has a security mode:

  • "full" ‚Äî bot can run anything (what Ryan uses)
  • "allowlist" ‚Äî bot can only run pre-approved commands
  • "deny" ‚Äî bot cannot run any commands

üìÅ read / write / edit ‚Äî File access

What it does: Lets the bot read, create, and modify files on your machine.

‚ö†Ô∏è Risk: MEDIUM. The bot can read/write files in its workspace by default. It needs this for memory files, notes, and projects.

Note: These are typically enabled by default within the workspace directory (~/.openclaw/workspace/).

üåê web_search / web_fetch ‚Äî Internet access

What it does: Lets the bot search the web and fetch content from URLs. Used for research, checking docs, pulling info.

‚ö†Ô∏è Risk: LOW. Read-only internet access. Bot can look things up but can't post or sign into anything.

üåç browser ‚Äî Browser automation

What it does: Lets the bot control a web browser ‚Äî click buttons, fill forms, take screenshots, navigate sites.

‚ö†Ô∏è Risk: MEDIUM-HIGH. The bot can interact with websites as if it were you. Only enable if you need it.

‚è∞ cron ‚Äî Scheduled tasks & reminders

What it does: Lets the bot set up recurring tasks ‚Äî reminders, daily checks, automated routines on a schedule.

‚ö†Ô∏è Risk: LOW. It just triggers messages/tasks at set times. Useful for morning briefings, reminders, etc.

üí¨ message ‚Äî Send messages to channels

What it does: Lets the bot proactively send messages to your Telegram/Discord/etc. without you asking first.

‚ö†Ô∏è Risk: MEDIUM. Bot can message you (or groups) on its own. Good for alerts and reminders. Make sure allowFrom in your channel config only includes YOUR user ID so strangers can't talk to your bot.

How to lock it down: In your Telegram channel config:

"dmPolicy": "allowlist",
"allowFrom": ["YOUR_TELEGRAM_USER_ID"]
To find your Telegram user ID: message @userinfobot on Telegram and it'll tell you.
